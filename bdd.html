Modèle conceptuel des données

Tout d’abord, nous avons dû réfléchir à l’agencement de notre base de données. Pour cela, nous avons choisi ensemble la meilleure solution possible, puis nous avons consulté Edwige CLERMONT et iUserLab afin qu’ils valident notre modèle ou pour nous faire des suggestions d’optimisation et d’amélioration..

Nous avons finalement choisi de créer 5 tables :

  • une pour les déficiences
  • une pour les catégories (et donc sous-catégories)
  • une pour les dispositifs
  • une pour les prix
  • et une pour les sites internet

comportant différentes informations, notamment un nom et une description (facultative) pour les déficiences, les catégories et les dispositifs.



Modèle logique des données

Ensuite, nous avons déterminé le modèle logique des données qui découle directement du modèle conceptuel et facilite a posteriori la création des différentes tables en SQL.



Création et implémentation des tables sur MySQL

Grâce au modèle logique des données, nous avons pû aisément créer les tables que nous avions déterminées préalablement. Pour cela, nous avons utilisé le langage de définition des données.

Enfin, une fois les 5 tables créées, il a fallu les remplir avec les données que nous avions recensées au premier semestre. Dans un premier temps, nous avons créé ensemble une « mini base de données » afin d’avoir un support sur lequel tester nos pages PHP. Dans un second temps, Marie a continué à implémenter la base de données pendant que les autres codaient le site web. Une fois la moitié des données implémentées, Émilie a pris le relais.

CREATE TABLE `DEFICIENCE`
(`id_deficience` INT(10) AUTO_INCREMENT,
`nom_def` VARCHAR(50) NOT NULL,
`texte_def`TEXT DEFAULT NULL,
PRIMARY KEY (`id_deficience`));

CREATE TABLE `SITE`
(`id_site` INT (10) AUTO_INCREMENT,
`nom_site` VARCHAR (100) DEFAULT NULL,
`url` VARCHAR (150) NOT NULL,
PRIMARY KEY(`id_site`));

CREATE TABLE `CATEGORIE`
(`id_categorie` INT(10) AUTO_INCREMENT,
`nom_cat` VARCHAR(100) NOT NULL,
`texte_cat` TEXT DEFAULT NULL,
`niveau` INT (3) DEFAULT NULL,
PRIMARY KEY (`id_categorie`),
`id_def_cat` INT(10) DEFAULT NULL,
FOREIGN KEY(`id_def_cat`) REFERENCES `DEFICIENCE`(`id_deficience`),
`id_cat_prec` INT(10) DEFAULT NULL,
FOREIGN KEY (`id_cat_prec`) REFERENCES `CATEGORIE`(`id_categorie`));

CREATE TABLE `DISPOSITIF`
(`id_dispositif` INT(10) AUTO_INCREMENT,
`nom_dis` VARCHAR(100) NOT NULL,
`description` LONGTEXT DEFAULT NULL,
`image` VARCHAR(100) DEFAULT NULL,
`id_cat_dis` INT(10) NOT NULL,
`id_site_fab` INT(10) NOT NULL,
PRIMARY KEY (`id_dispositif`),
FOREIGN KEY (`id_cat_dis`) REFERENCES `CATEGORIE` (`id_categorie`),
FOREIGN KEY (`id_site_fab`) REFERENCES `SITE` (`id_site`));

CREATE TABLE `PRIX` (`id_prix` INT(10) AUTO_INCREMENT,
`prix` TEXT DEFAULT NULL,
`id_dis_prix` INT(10) NOT NULL,
`id_site_vente` INT(10) NOT NULL,
PRIMARY KEY(`id_prix`),
FOREIGN KEY (`id_site_vente`) REFERENCES `SITE` (`id_site`),
FOREIGN KEY (`id_dis_prix`) REFERENCES `DISPOSITIF`(`id_dispositif`));
